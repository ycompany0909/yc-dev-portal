## 6. 成長余地（伸びしろ）

### ① Claude 非依存の独立実行力

414 / 414 commits が Claude 協働。これは強みでもあるが、「Claude なしで 1 ファイル書く力」のキャリブレーションができていない。例えば：
- Cloudflare Workers の Service Binding をドキュメント読みながら設定できるか
- D1 のスキーマを SQL だけで設計できるか
- ブラウザコンソールでデバッグしてバグを言語化できるか

### ② 単体テストの不在

CI 側のスモークテスト 43 件は素晴らしいが、Pages Function / Worker の関数単位テストがない。リファクタ時の安全網が弱い。**Vitest** 等の導入余地。

### ③ DB スキーマ・API 仕様の図示

13 個の Pages Functions、8 本の Workers、複数の D1 テーブルが頭の中にしかない（CLAUDE.md の影響マップは粗い粒度）。**スキーマ図** や **API リファレンス** を自動生成する仕組みがあれば、新しい人を巻き込みやすい。

### ④ リファクタ比率の低さ

414 commits 中 refactor 11 件（3%）。フィットしているうちは良いが、お客様 13,860 件・全 8 Worker を「保守し続ける」フェーズに入ったら、定期的なコード整理が必要。

### ⑤ 新人 / 外部協力者の巻き込み

カスタムビューでロール分離はできているが、「Claude セッションを始めて 5 分で何ができるか」のオンボーディング動線がない。CLAUDE.md は濃いが入門資料としては重い。
