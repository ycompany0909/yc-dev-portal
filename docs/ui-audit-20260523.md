## 6. display_order 全て 999 問題

breadcrumb.json で全 69 ページの `display_order: 999` → ソート不能。
これは breadcrumb-editor.js（5/23 リファクタで作った編集 UI）が `display_order` を設定する仕組みになっているが、**運用が回ってない**。

→ 整理時に `display_order` を `1, 2, 3...` の連番で設定すれば、サイドバーがアルファベット順以外で並ぶ。

## 8. 実装手順案

### Phase 1: 整理計画レビュー（オルズグルさん作業）

1. このレポートをレビュー
2. カテゴリ再編案を OK / 修正
3. **重複ページの統合判断**（ナーチャリング 3 ページ → 1 ページ等）

### Phase 2: breadcrumb.json 一括更新

```json
// 各ページに category と display_order を再設定
{
  "asakai.html": {
    "category": "admin",
    "display_order": 1,
    "title": "朝会管理"
  },
  ...
}
```

→ サイドバーが新カテゴリ構成で再表示される。

### Phase 3: index.html のセクション再編

- `<h2 data-category="admin">` の対応する HTML セクションも再配置
- 必要なら h2 ラベルも変更
