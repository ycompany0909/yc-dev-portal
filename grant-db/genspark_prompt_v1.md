## Gensparkへの指示方針

**Gensparkの強みを最大活用する設計：**
- ウェブ検索 + 複数ページ閲覧 + 情報合成 → 得意
- コード実行・ファイル出力・API呼び出し → 不可
- 出力形式：Sparkpage内のMarkdownテーブル・JSONブロック（NocoDB貼り付け用）

**現実的な目標：**
- Phase A（今回）：全財団の名前・URL・基本情報リストを完成させる（骨格）
- Phase B：上位100財団の詳細情報・採択実績を収集する
- Phase C：評判シグナルを収集する

### B-2. ゲートサイトの迂回取得

マイページ登録が必要で直接閲覧できない財団については、以下の迂回戦略を試してください：

**迂回ルートA：都道府県・自治体HP経由**
```
検索: "[財団名] 助成金 要項 2026 site:pref.OR.site:city"
例: "ニッセイ財団 助成金 申請 2026 県"

```

## Phase C：評判・実態シグナルの収集

### C-1. NPOコミュニティの声

以下の検索を実施し、財団への評判情報を収集してください：

**検索クエリ群：**
```
"助成金 申請 してみた" site:note.com
"○○財団 採択 ました" site:note.com OR site:ameblo.jp
"助成金 申請 落ちた 理由"
"財団 助成金 申請 大変 面倒"
"助成金 おすすめ 財団 NPO"
"財団 助成金 返答 遅い 問い合わせ"
"助成金 報告書 大変 NPO 負担"
```

各記事・投稿から以下を取得：

| 財団名 | 情報種別 | 投稿日 | センチメント | トピック | 引用（原文） | 情報源URL |
|---|---|---|---|---|---|---|

**センチメント基準：**
- positive：「また申請したい」「わかりやすい」「フィードバック丁寧」
- negative：「返答遅い」「要件不明確」「落選理由不明」「もう申請しない」
- neutral：事実記述のみ
- mixed：良悪混在

## 作業上の重要な注意事項

1. **全ページネーションを辿ること**：1ページ目だけで止まらない
2. **重複を排除すること**：複数ソースに同じ財団が出たらマージする
3. **年度を必ず明記**：2025年度・2026年度・不明 を区別する
4. **金額は円単位の整数**：「100万円」→ 1000000
5. **締め切りは月のみでもOK**：正確な日付が不明な場合は「11月」等
6. **情報源URLを全件記録**：どこから取ったか必ず残す
7. **不明な場合はnullと記録**：推測で埋めない
8. **採択実績は年度付きで**：古いデータと新しいデータを混在させない
